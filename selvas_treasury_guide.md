## 1. 시스템 개요

셀바스 통합자금관리 시스템은 3개 법인(셀바스에이아이·셀바스헬스케어·메디아나)의 자금 현황을 **하나의 화면에서 실시간으로 통합 관리**하는 웹 기반 시스템입니다.

| 항목 | 내용 |
|------|------|
| 접속 URL | `https://yeansoojeong.github.io/treasury/` |
| 지원 환경 | PC(Chrome/Edge 권장), 모바일(Safari/Chrome) |
| 데이터 저장 | Supabase 클라우드 DB — 어느 기기에서든 동일 데이터 |
| 실시간 환율 | ExchangeRate-API 자동 조회 |

## 2. 접속 방법

### PC

1. 브라우저에서 접속 URL 입력
2. 접속 코드 입력 후 **로그인** 버튼 클릭
3. 로그인 성공 시 통합 상황판 자동 이동

### 모바일

1. 브라우저에서 접속 URL 입력 (홈 화면에 추가 권장)
2. 접속 코드 입력 후 로그인
3. 화면 하단 탭바로 메뉴 이동

> **팁**: 모바일에서 홈 화면에 추가하면 앱처럼 사용할 수 있습니다.
> - iOS: Safari → 공유 → 홈 화면에 추가
> - Android: Chrome → 메뉴(⋮) → 홈 화면에 추가

## 3. 접속 코드 및 권한

### 권한 등급

| 등급 | 접근 범위 | 가능한 작업 |

## 5. 통합 상황판 (대시보드)

### 요약 카드 (상단 4개)

| 카드 | 내용 |
|------|------|
| 운전 + 운용 합계 | 전체 자금 규모 |
| 원화 자금 합계 | 보통예금/MMDA + 국책 + CMA 등 (가용/불가용 분리) |
| 외화 원화환산액 | USD·EUR·JPY 환산 합계 (가용/불가용 분리) |
| 원화/외화 비율 | 비율 바 그래프 |

### 지표 카드 (클릭 가능 — 4개)

카드를 클릭하면 하단 차트와 우측 상세 목록이 해당 항목으로 변경됩니다.

| 카드 | 색상 | 내용 |
|------|------|------|
| 운전자금 | 파랑 | 보통예금·국책·CMA 합계 |
| 운용자금 | 초록 | 단기투자 상품 합계 |
| 차입금 | 노랑 | 대출·차입 합계 |
| 순현금 | 보라 | 운전+운용−차입금 |

> **카드 색상 테두리**로 클릭 가능 여부를 구분합니다. 마우스를 올리면 카드가 살짝 확대됩니다.

### 차트 영역

- 기간 선택: 7일 / 30일 / 90일 / 1년
- 모바일에서는 차트만 표시 (상세 목록 숨김)

### 알림 배너

| 색상 | 의미 |
|------|------|
| 🔴 빨강 | 오늘 운전자금 미입력 |
| 🟡 노랑 | 직전 영업일 미입력 |
| 🔵 파랑 | 지분투자 평가금액 미업데이트 |

## 6. 운전자금 입력

### 입력 항목

| 항목 | 설명 |
|------|------|
| 법인 | 입력 대상 법인 선택 |
| 날짜 | 기준일 (오늘 날짜 자동 입력) |
| 작성자 | 입력자 이름 |
| 보통예금/MMDA | 원화 보통예금·MMDA 잔액 |
| 국책자금 | 정책금융 등 |
| 증권(CMA) 예수금 등 | CMA·증권 예수금 |
| USD/EUR/JPY | 외화 잔액 (해당 통화 기준) |
| 메모 | 특이사항 |

### 기존 데이터 수정

하단 이력 목록에서 **행을 클릭**하면 해당 데이터가 위 폼에 자동으로 불러와집니다. 수정 후 **"운전자금 수정 저장"** 버튼을 클릭합니다.

### 저장 확인

저장 성공 시 상단 topbar에 **☁ 저장됨** 표시가 3초간 나타납니다.

## 7. 운용자금 (단기투자)

### 입력 항목

| 항목 | 설명 |
|------|------|
| 법인 | 운용 주체 법인 |
| 금융기관/발행처 | 운용 기관명 |
| 상품명 | 예금·채권 등 상품명 |
| 수익률 (%) | 연 이자율 |
| **가용여부** | 가용(즉시 사용 가능) / 불가용(만기 전 인출 불가) |
| 통화/금액 | 운용 금액 |
| 가입일/만기일 | 기간 설정 |

### 가용/불가용 구분

- 대시보드 원화/외화 카드에서 가용·불가용 금액을 분리해서 표시합니다.
- 목록에서 초록(가용) / 빨강(불가용) 배지로 확인할 수 있습니다.

### 기존 데이터 수정

목록에서 행 클릭 → 폼에 자동 로드 → 수정 후 **"수정 저장"** 클릭

## 8. 차입금 (대출)

### 입력 항목

| 항목 | 설명 |
|------|------|
| 법인 | 차입 법인 |
| 차입처 | 금융기관명 |
| 구분 | 담보대출·신용대출·정책자금 등 |
| 통화/금액 | 차입 금액 |
| 이자율 (%) | 연 이자율 |
| 실행일/만기일 | 차입 기간 |

### 상환 처리

목록에서 **상환** 버튼 클릭 → 확인 팝업 → 목록에서 삭제 및 대시보드 반영

### D-Day 표시

만기일까지 남은 일수를 표시합니다. 만기 경과 시 빨간색으로 표시됩니다.

## 9. 지분투자 관리

### 입력 항목

| 항목 | 설명 |
|------|------|
| 보유법인 | 지분 보유 법인 |
| 기준일 | 평가 기준일 |
| 종목명 | 투자 회사명 |
| 티커 | 주식 코드 (상장 종목) |
| 투자목적 | 전략적 투자·재무적 투자 등 |
| 상장여부 | KOSPI·KOSDAQ·비상장 |
| **가용여부** | 가용(처분 가능) / 불가용(처분 제한) |
| 보유 주식수 | 주 |
| 1주당 단가 | 원 |

### 평가금액 자동 계산

주식수 × 단가 = 총 평가금액 자동 계산

### 시세 자동 업데이트

상장 종목은 **시세↑** 버튼으로 실시간 주가를 불러올 수 있습니다.

### 히스토리 보존 방식

동일 종목을 여러 번 저장하면 모든 이력이 Supabase에 누적 보존됩니다. 화면에는 **가장 최신 날짜 기준** 데이터만 표시됩니다.

---

## 10. 자금 변동 이력

### 조회 방법

- 기간(시작일~종료일) 설정 후 **조회** 버튼 클릭
- 운전자금 입력 기록을 기준으로 해당 날짜의 운용자금·차입금·지분투자·순현금을 함께 표시

### 컬럼 구성

| 컬럼 | 설명 |
|------|------|
| 일자 | 운전자금 입력 기준일 |
| 법인 | 입력 법인 |
| 원화운전 | 보통예금+국책+CMA 합계 |
| USD·EUR·JPY | 외화 잔액 |
| 외화환산 | 외화 원화환산 합계 |
| 운용자금 | 해당일 기준 운용자금 잔액 |
| 차입금 | 해당일 기준 차입금 잔액 |
| 지분투자 | 해당일 이전 최신 평가금액 합계 |
| 순현금 | 운전+운용−차입금 |
| 비고 | 메모 |

> **주의**: 이력은 운전자금 입력 데이터를 기준으로 생성됩니다. 운전자금을 입력하지 않은 날은 이력이 표시되지 않습니다.

## 11. 환율 현황

### 자동 업데이트

- 로그인 시 자동으로 최신 환율 조회
- **↺ 환율** 버튼으로 수동 갱신 가능
- 출처: ExchangeRate-API (사이드바 하단에 출처 표시)

### 평가금액 업데이트

**전체 평가금액 업데이트** 버튼 클릭 시 현재 환율 기준으로 모든 외화 자산의 원화환산액을 재계산합니다.

## 12. 관리 메뉴 (Master 전용)

Master 계정으로 로그인 시 좌측 사이드바 하단에 **관리** 섹션이 표시됩니다.

### 내 코드 변경

Master 계정 본인의 접속 코드를 변경합니다.

1. 현재 코드 입력
2. 새 코드 입력 (6자 이상)
3. 새 코드 확인 재입력
4. **코드 변경 저장** 클릭

### 사용자 코드 관리

모든 사용자의 접속 코드를 조회·수정·비활성화할 수 있습니다.

| 기능 | 방법 |
|------|------|
| 전체 목록 조회 | 메뉴 진입 시 자동 로드 |
| 코드 초기화 | 해당 행의 **초기화** 버튼 클릭 |
| 코드 비활성화 | 해당 행의 **비활성화** 버튼 클릭 |
| 새 사용자 추가 | 우상단 **+ 코드 추가** 버튼 |

### 데이터 관리

| 기능 | 설명 |
|------|------|
| localStorage → Supabase 이전 | 이전 방식 데이터를 클라우드로 이전 |
| JSON 백업 다운로드 | 전체 데이터를 JSON 파일로 내보내기 |
| JSON 파일 복원 | 백업 파일로 데이터 복원 |
| 연결 재확인 | Supabase 연결 상태 및 데이터 건수 확인 |
| **전체 데이터 초기화** | Supabase의 모든 자금 데이터 영구 삭제 (마스터 코드 재입력 필요) |

> ⚠ **전체 초기화는 복구 불가능합니다.** 반드시 JSON 백업 후 실행하세요.

## 14. 데이터 저장 구조

### Supabase 클라우드 DB

모든 데이터는 Supabase PostgreSQL에 저장됩니다. 어느 기기·브라우저에서 접속해도 동일한 데이터를 공유합니다.

### 테이블 구성

| 테이블 | 저장 내용 |
|--------|----------|
| `daily` | 운전자금 일일 입력 이력 |
| `investments` | 운용자금(단기투자) 현황 |
| `loans` | 차입금(대출) 현황 |
| `equities` | 지분투자 평가 이력 (전체 누적) |
| `price_history` | 주가·기준가 이력 |
| `access_codes` | 접속 코드 및 권한 관리 |

### 저장 시점

- 데이터 입력 후 **저장/추가** 버튼 클릭 시 즉시 Supabase에 저장
- 저장 성공: topbar에 **☁ 저장됨** (초록, 3초)
- 저장 실패: **⚠ 저장 실패** (빨강, 3초) — 네트워크 재확인 필요

### 정기 백업 권장

Master → 데이터 관리 → **JSON 백업 다운로드** 를 월 1회 이상 실행하세요.

## 15. 자주 묻는 질문 (FAQ)

**Q. 다른 PC에서 접속하면 데이터가 없어요.**

A. Supabase 연동이 완료된 이후에는 어느 기기에서나 동일한 데이터가 표시됩니다. 처음 접속 시 상단에 파란 로딩 배너가 표시되며 데이터를 불러옵니다. 만약 데이터가 보이지 않으면 Master 계정으로 데이터 관리 → 연결 재확인을 실행해 주세요.

**Q. 저장됨이 뜨지 않고 저장 실패가 떠요.**

A. 인터넷 연결을 확인하세요. 연결이 정상이라면 Master → 데이터 관리 → 연결 재확인으로 Supabase 상태를 점검하세요.

**Q. 상환을 눌렀는데 차입금이 남아있어요.**

A. 확인 팝업에서 반드시 **확인**을 눌러야 삭제됩니다. 삭제 후 대시보드에서 변경 내역을 확인하세요.

**Q. 운전자금을 잘못 입력했어요.**

A. 하단 이력 목록에서 해당 행을 클릭하면 폼에 불러와집니다. 수정 후 **수정 저장** 버튼을 클릭하세요.

**Q. 모바일에서 화면이 너무 작아요.**

A. 브라우저 설정에서 확대 비율을 조정하거나, 핀치 줌으로 특정 영역을 확대해서 보세요.

**Q. 접속 코드를 분실했어요.**

A. Master 계정에 문의하세요. Master → 관리 → 사용자 코드 관리에서 코드를 재설정할 수 있습니다.

**Q. 지분투자 평가금액이 이력에 안 나와요.**

A. 자금 변동 이력은 운전자금 입력 기록을 기준으로 생성됩니다. 해당 날짜에 운전자금을 입력해야 지분투자 금액도 이력에 표시됩니다.
